Before deployment

Application checklist

  1. Fix DNS: add explicit records or wildcard DNS for *.kafka.apps.sub.comptech-lab.com to the HAProxy IP.
  2. Create app identity: do not reuse platform Kafka users such as mm2 or redis-applier.
  3. Define topics and ACLs: model least privilege for the app producer and consumer behavior.
  4. Handle secrets safely: use the validated Vault/ESO pattern; runtime credentials must not be committed into Git.
  5. Mount trust: provide the Strimzi cluster CA to the Java trust configuration.
  6. Choose placement: start on spoke-dc, then decide whether spoke-dr is active standby.

Runtime shape

Expected OCP resources

  • Namespace with explicit owner labels and optional istio.io/dataplane-mode=ambient label.
  • Deployment for the Java/JBoss container with readiness and liveness probes.
  • ConfigMap for non-secret Kafka bootstrap and app tuning.
  • Secret reference for SCRAM username/password from an app-specific ExternalSecret backed by Vault.
  • Service and Route or Istio Gateway/VirtualService, depending on whether mesh routing is in scope.
  • NetworkPolicy only after required egress to Kafka and platform services is clear.
