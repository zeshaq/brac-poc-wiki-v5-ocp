A focused view of the OpenShift fleet, setup details, current state, and roadmap. The RKE2 platform has its own catalogue; this wiki is scoped to the OCP clusters and their GitOps-managed platform services.

Resilience

Backup and DR

  1. Backup freshness gate: create or wait for a fresh ACM hub backup that satisfies the agreed drill RPO.
  2. Image readiness: implement ACM/MCE image mirror or pre-pull so hub-dr can start quickly during site failure.
  3. Runbook validation: dry-run restore manifests and confirm passive hub state before promotion.
  4. Controlled activation: run the hub-dc to hub-dr activation drill only after all gates are clean.

General OADP daily backups completed on all four clusters in the latest recorded run. The ACM hub backup freshness remains a drill gate because a strict two-hour RPO can expire before the operator starts the drill.

Governance

Operating rules

  • Do not print secrets, kubeconfigs, pull secrets, kubeadmin passwords, or PAT values.
  • Use read-only checks before remediation so actions are not based on stale state.
  • Treat lab-gitops-full/ as platform desired state and lab-workloads/ as the workload repo.
  • Record live checks, live changes, file edits, validation, and residual risk in session reports.
  • Keep historical state explicit. Do not rediscover from scratch unless recorded state is stale or contradicted by live checks.
