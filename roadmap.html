Work tracking

GitHub issues and project board

  • Use GitHub Issues for bounded OpenShift tasks, risks, decisions, and DR drills.
  • Use milestones for major execution goals: hub DR, durable image mirror, ODF Regional-DR, and application onboarding.
  • Use the GitHub Project board for queue status. Keep implementation details in GitOps repositories and session evidence in the local workspace reports.
  • Do not store secrets, kubeconfigs, kubeadmin passwords, pull secrets, PAT values, or full Secret manifests in GitHub tracking objects.
